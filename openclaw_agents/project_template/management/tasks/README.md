# Tasks

Create one file per execution task in this folder.

Recommended naming:
- `TASK-001-short-title.md`
- `TASK-002-short-title.md`

Each task should:
- map back to one story
- define the concrete execution target
- include definition of done
- record validation expectations
- move through clear statuses such as `draft`, `ready`, `in_progress`,
  `blocked`, `done`, and `verified`
