# Releases

## Purpose
Prepare and validate a change set for delivery.

## Trigger
Use this skill when packaging work for handoff, deployment, or release notes.

## Inputs
- Scope of released changes
- Required checks
- Versioning or release process, if defined

## Steps
1. Confirm the release scope and any included risks.
2. Run the required quality gates.
3. Summarize user-visible changes and operational impact.
4. Verify any version, artifact, or packaging expectations.
5. Document rollback or recovery considerations when relevant.

## Verification
- Required checks are green
- Release notes or summary match the actual changes
- Known risks and follow-ups are documented

## Notes
- Keep release communication factual and concise.
- Do not claim verification that was not actually performed.
